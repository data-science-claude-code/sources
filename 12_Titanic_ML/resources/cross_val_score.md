# 교차검증(Cross Validation)을 하는 이유

**모델의 성능을 더 신뢰할 수 있게 측정**하기 위해서입니다.

---

## 문제 상황 — train/test 단순 분할의 한계

```
전체 데이터 891개를 80/20으로 나누면
→ 테스트 세트 179개만으로 성능을 판단
→ 운 좋게 쉬운 샘플이 테스트에 몰리면 과대평가
→ 운 나쁘면 과소평가
```

## K-Fold 교차검증은 이를 해결

```
5-Fold 예시:
[Fold1] ████ ████ ████ ████ [TEST] → Accuracy: 0.82
[Fold2] ████ ████ ████ [TEST] ████ → Accuracy: 0.80
[Fold3] ████ ████ [TEST] ████ ████ → Accuracy: 0.83
[Fold4] ████ [TEST] ████ ████ ████ → Accuracy: 0.81
[Fold5] [TEST] ████ ████ ████ ████ → Accuracy: 0.79

평균: 0.81 ± 0.014  ← 이게 진짜 성능
```

모든 데이터가 한 번씩 테스트에 사용되므로 **특정 분할에 의한 운을 제거**할 수 있습니다.

---

## 표준편차도 중요한 이유

| 모델 | 평균 | 표준편차 |
|------|------|---------|
| A | 0.83 | 0.01 | ← 안정적 |
| B | 0.83 | 0.08 | ← 불안정 (데이터에 민감) |

평균이 같아도 표준편차가 크면 **새 데이터에서 성능이 들쑥날쑥**할 수 있습니다.
