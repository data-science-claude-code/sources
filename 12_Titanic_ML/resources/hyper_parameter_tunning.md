# 하이퍼파라미터 튜닝 원리

**핵심 개념:** 모델이 스스로 학습하는 값(파라미터)이 아니라, **사람이 직접 설정해야 하는 값**을 최적화하는 과정입니다.

---

## 파라미터 vs 하이퍼파라미터

| | 파라미터 | 하이퍼파라미터 |
|--|---------|--------------|
| 누가 결정? | 모델이 학습으로 스스로 결정 | 사람이 직접 설정 |
| 예시 | 회귀 계수, 가중치 | 트리 깊이, 트리 개수 |

---

## GridSearchCV 원리 — 모든 조합을 다 시도해보기

```python
param_grid = {
    'n_estimators': [100, 200],
    'max_depth':    [3, 5, 10],
}
```

위 설정이면 2 × 3 = **6가지 조합**을 모두 시도합니다.

```
조합 1: n_estimators=100, max_depth=3  → CV 점수: 0.81
조합 2: n_estimators=100, max_depth=5  → CV 점수: 0.83
조합 3: n_estimators=100, max_depth=10 → CV 점수: 0.82
조합 4: n_estimators=200, max_depth=3  → CV 점수: 0.82
조합 5: n_estimators=200, max_depth=5  → CV 점수: 0.84  ← 최고
조합 6: n_estimators=200, max_depth=10 → CV 점수: 0.83

best_params_ = {'n_estimators': 200, 'max_depth': 5}
```

각 조합마다 **5-Fold CV**로 점수를 측정하고, 가장 높은 조합을 선택합니다.

## RandomForestClassifier 주요 하이퍼파라미터 설명

### n_estimators (트리 개수)

랜덤 포레스트는 여러 결정 트리를 만들어 **다수결로 예측**합니다.

```
n_estimators=3 일 때:
트리1 → 생존  ┐
트리2 → 사망  ├→ 다수결 → 생존 (2:1)
트리3 → 생존  ┘

n_estimators=200 일 때: 200개 트리의 다수결 → 더 안정적
```

- 많을수록 성능 안정 / 학습 시간 증가
- 이번 실험: `[100, 200]`

---

### max_depth (트리 최대 깊이)

결정 트리가 질문을 몇 단계까지 할 수 있는지 제한합니다.

```
max_depth=2:              max_depth=5:
   성별?                     성별?
  ↙    ↘                   ↙    ↘
여성   남성?             여성    남성?
      ↙   ↘                    ↙   ↘
    1등?  사망            나이<10?  사망
    ...                  ↙    ↘
                       생존   등급?
                              ...
```

- 깊을수록 훈련 데이터에 과적합(overfitting) 위험
- 얕을수록 과소적합(underfitting) 위험
- 이번 실험: `[3, 5, 10, None]`

---

### min_samples_split (분기 최소 샘플 수)

노드를 나누려면 최소 몇 개의 샘플이 있어야 하는지 설정합니다.

```
min_samples_split=2:          min_samples_split=5:
샘플 2개만 있어도 분기 가능    샘플 5개 미만이면 분기 중단
→ 트리가 세밀하게 학습         → 트리가 단순하게 학습
→ 과적합 위험                  → 과적합 방지
```

- 값이 클수록 트리가 단순해져 과적합 억제
- 이번 실험: `[2, 5]`

---

### 세 파라미터의 관계 요약

| 파라미터 | 크면 | 작으면 |
|---------|------|-------|
| `n_estimators` | 안정적, 느림 | 불안정, 빠름 |
| `max_depth` | 과적합 위험 | 과소적합 위험 |
| `min_samples_split` | 단순한 트리 | 복잡한 트리 |
