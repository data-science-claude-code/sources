# TEP (Tennessee Eastman Process) 데이터셋 설명

## 1. 개요

TEP는 **테네시 이스트만 공정(Tennessee Eastman Process)** 의 약자로, 1993년 Downs와 Vogel이 공정 제어 및 고장 탐지 연구를 위해 설계한 **가상의 화학 공정 시뮬레이션 데이터셋**입니다. 실제 화학 공장의 공정 데이터를 모사한 벤치마크 데이터셋으로, 전 세계 공정 모니터링·이상 탐지(Fault Detection and Diagnosis, FDD) 연구에 가장 널리 사용됩니다.

---

## 2. 공정 구조

TEP는 5개의 주요 단위 공정(Unit Operation)으로 구성됩니다:

| 단위 공정 | 설명 |
|----------|------|
| 반응기 (Reactor) | 4가지 반응물(A, C, D, E)이 반응하여 제품(G, H)과 부산물(F) 생성 |
| 응축기 (Condenser) | 반응기 출구 가스를 냉각 |
| 기체-액체 분리기 (Separator) | 기체와 액체 혼합물 분리 |
| 압축기 (Compressor) | 재순환 가스를 압축 |
| 스트리퍼 (Stripper) | 제품 정제 |

## 5. 데이터셋 파일 구성

본 프로젝트에서 사용하는 4개의 CSV 파일:

| 파일 | 시뮬레이션 수 | 샘플 수 | 설명 |
|------|-------------|---------|------|
| `TEP_FaultFree_Training.csv` | 500 | 500 × 500 스텝 | 정상 운전 훈련 데이터 |
| `TEP_FaultFree_Testing.csv` | 100 | 100 × 960 스텝 | 정상 운전 테스트 데이터 |
| `TEP_Faulty_Training.csv` | 2,100 | 21 고장 × 100회 × 500 스텝 | 고장 유형별 훈련 데이터 |
| `TEP_Faulty_Testing.csv` | 2,100 | 21 고장 × 100회 × 960 스텝 | 고장 유형별 테스트 데이터 |

- **샘플링 간격**: 3분
- **훈련 데이터**: 500 스텝 × 3분 = 25시간 (고장은 20번째 스텝부터 주입)
- **테스트 데이터**: 960 스텝 × 3분 = 48시간 (고장은 160번째 스텝부터 주입)

## 6. 컬럼 구조

각 CSV는 다음 컬럼을 포함합니다:

| 컬럼 | 설명 |
|------|------|
| `simulationRun` | 시뮬레이션 실행 번호 |
| `sample` | 시계열 스텝 번호 |
| `xmeas_1` ~ `xmeas_41` | 측정 변수 41개 |
| `xmv_1` ~ `xmv_11` | 조작 변수 11개 |
| `faultNumber` | 고장 번호 (0=정상, 1~21=고장 유형) |

## 7. 머신러닝 태스크

TEP 데이터로 수행하는 주요 ML 태스크:

| 태스크 | 설명 |
|--------|------|
| **고장 탐지 (Fault Detection)** | 정상(0) vs 고장(1) 이진 분류 |
| **고장 진단 (Fault Diagnosis)** | 21가지 고장 유형 다중 분류 |
| **고장 예측 (Fault Prediction)** | 고장 발생 전 조기 감지 |

본 프로젝트(`16_TEP_Fault_ML`)는 **고장 진단(22-class 다중 분류)** 에 초점을 맞춥니다.
