# 파생 피처(Derived Feature)는 ML 성능 향상에 필수인가?

## 결론: 필수는 아니다

파생 피처는 도구 중 하나일 뿐이고, 상황에 따라 효과가 크게 다르다.

---

## 파생 피처가 효과적인 경우

| 상황 | 예시 |
|------|------|
| 원본 피처 간 **비선형 관계**가 타겟과 연관될 때 | BMI = Weight / Height² |
| 도메인 지식으로 **의미 있는 조합**을 알 때 | 금융에서 부채비율 = 부채 / 자산 |
| 원본 피처만으로 모델이 관계를 학습하기 어려울 때 | 트리 계열은 나눗셈 관계를 근사하기 어려움 |

---

## 파생 피처 없이도 성능이 좋은 경우

| 상황 | 이유 |
|------|------|
| **딥러닝** | 층이 깊어질수록 내부적으로 복잡한 조합을 스스로 학습 |
| **RBF SVM, Random Forest, XGBoost** | 비선형 경계를 자체적으로 포착하는 능력이 강함 |
| 피처가 이미 **충분히 표현력** 있을 때 | 원본 피처만으로도 패턴이 명확한 경우 |

> BMI 태스크에서 파생 피처 없이 **SVM(RBF) F1 = 0.89** 달성

---

## 실무에서 파생 피처를 쓰는 진짜 이유

성능 향상보다 **학습 속도·데이터 효율 개선**이 목적인 경우가 많다.

- **데이터가 적을 때**: 모델이 관계를 학습할 샘플이 부족 → 공식으로 직접 넣어줌
- **선형 모델 사용 시**: Logistic Regression처럼 비선형 표현이 불가한 모델 보완
- **해석 가능성**: 피처 자체가 의미를 가져야 설명이 쉬운 경우

---

## 한 줄 요약

> 파생 피처는 **도메인 지식 + 부족한 데이터 + 단순한 모델** 조합일 때 효과가 크고,
> 강력한 모델(XGBoost, SVM, 딥러닝)과 충분한 데이터가 있다면 없어도 된다.
