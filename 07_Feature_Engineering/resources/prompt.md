# 피처 엔지니어링 프롬프트

`customer_churn_raw.csv`라는 고객 이탈 데이터셋이 있습니다. 고객이 이탈할지 여부를 예측하려고 합니다.

모델을 구축하기 전에 피처 엔지니어링을 수행하세요:

1. 기존 피처와 이탈(churn) 타겟 변수 간의 관계 분석
2. 예측 성능을 높일 수 있는 새로운 파생 피처를 최소 8개 이상 생성. 다음 항목을 고려하세요:
   - 기존 피처 간의 비율(ratio) 및 상호작용(interaction)
   - 행동 지표 (Behavioral indicators)
   - 임계값 기반 위험 플래그 (Risk flags)
   - SaaS 비즈니스에 적합한 도메인 특화 피처
3. 각 새로운 피처마다 예측에 유효하다고 생각하는 이유 설명
4. 각 새로운 피처와 이탈 타겟 간의 상관관계 출력
5. 보강된 데이터셋을 `customer_churn_engineered.csv`로 저장
