# 공식 계산값 vs AI 예측값 — 차이 분석

## 결론: 틀릴 수 있다 (특히 경계값 근처에서)

---

## 왜 틀릴 수 있는가?

**공식 계산값**은 BMI 수치를 정해진 기준으로 자르는 **결정론적(deterministic)** 방법이다.

```
BMI = 24.9 → Normal (Index 2)  ← 공식은 항상 정확
BMI = 25.0 → Overweight (Index 3)
```

**AI 예측값**은 Height·Weight를 보고 경계선을 **근사(approximation)** 학습한 것이다.
BMI 공식을 모르고, 500개 샘플에서 패턴을 추론했다.

```
Height=175, Weight=76 → BMI 24.8 (Normal)
                       → AI가 Overweight 로 잘못 예측할 수 있음
```

---

## 실제로 얼마나 틀리나?

| 클래스 | Recall | 의미 |
|--------|:------:|------|
| Normal | **0.71** | 실제 Normal 14개 중 4개를 다른 클래스로 오예측 |
| Overweight | 0.93 | 실제 Overweight 14개 중 1개 오예측 |

전체 Test 100개 중 **9개** (= 1 - 0.91 Accuracy)는 공식 결과와 AI 예측이 불일치한다.

---

## 가장 많이 틀리는 구간

```
BMI 경계 근처
  24.9 ↔ 25.0  (Normal ↔ Overweight)
  29.9 ↔ 30.0  (Overweight ↔ Obesity)
```

Height·Weight 두 숫자만 보고 이 미세한 경계를 구분하는 건 AI에게 어렵다.
이것이 BMI 파생 피처 없이 학습하는 **진짜 어려움**이고,
SVM(RBF)의 F1이 1.0이 아닌 **0.89**인 이유다.

## 앱에서 두 값의 위치

| 위치 | 값 | 방식 |
|------|-----|------|
| 왼쪽 입력 패널 — "참고 BMI (공식 계산)" | 공식 계산값 | `Weight ÷ Height(m)²` |
| 오른쪽 결과 패널 — 게이지 빨간 선 | 동일한 공식 계산값 | 게이지 표시용 |
| 오른쪽 결과 패널 — 비만 단계 텍스트 | **AI 예측값** | SVM(RBF) 모델 예측 |
